Java OOPs concept

In this page, we will learn about the basics of OOPs. Object-oriented programming is a paradigm that provides many concept, such as inheritence, data biding,polymophism, etc.

Simula is considered the first object-oriented programming language.The programming paradigm where everything is represented as an object is known as a truly object-oriented programming language.

The popular object-oriented language java, #c, PHP, Python, C++ etc

The main aim of object-oriented programming is to implement real-world entities, for example, object,classes, abstraction,inheritence,polymophism etc.

OOps (Object-oriented programming)

Object means a real-world entity such as a pen, chair, table,computer,watch etc. Object-oriented programming is a methodology or paradigm to design a program using classes and object. It simplifies software development and maintenance by providing some concept.

1. Object

2. class

3. Inheritence

4. polymophism

5. Abstraction

6. Encapsulation

Object

Any entity that has state and behaviour is known as an object. for example, a chair, pen, bike, etc. It can be physical or logical.

An object can be defined as an instance of a class. An object contains an address and takes up some space memory.

cmmunicate without knowing the details of each other's data or code. The only necessary thing is the type of message and the type of response returned by the object.

Example: A dog is an object because it has state like color, name, breed , etc.. as well as behavious like wagging the tail, braking, eating etc.

Class

Collection of object is called class. It is a logical entity.

A class can also be defined as a blueprint from which you can create an individual object. Class doesn't consume any space.
Polymophism
If one task is performed in many ways, it is known as polymorphism. For example, shape, triangle, reactangle, etc.

In java, we use method overloading and method overriding to achieve polymorphism.

Another example can be to speak something; for example, a cat speaks meow, dog barks woof,etc
Abstraction
Hiding internel datail and showing functionality is known as abstraction. For example phone call, we don't know the internel processing.

In java, we use abstract class and interface to achieve anstraction.

 Encapsulaiton
Binding (or wrapping) code and data together into a single unit are known as Encapsulation. for example, a capsule, it is wrapped with different medicines.

A Java class is the example of Encapsulation. Java bean is the fully encapsulated class because all the data members are private here.

 Coupling
Coupling refers to the knowlege or imformtion or dependency of another class. It arises when classes are aware of each other. If a class has the details information of another class, there is strong coupling. In java, we use private, protected and public modifiers todisplay the visibility level of a class, method, and field. you can use interface for the weaker coupling because there is no concrete implementtion.

 Cohesion
Cohesion refers to the levels of a component which performs a single well-defined task. A single well-defined task is done by a highly cohesive method. The weekly cohesive method will split the task into seprate parts. The java.io package is a highly cohesive package beacuse it has I/O related classes and interface. however, The java.util package because it has unrelated classes and interface.

 Association
Association represents the relationship between the object. Here, one object can be associated with one object or many objects.There can be four types of associated with one object or many objects. There can be four types of Association between object:

1. One to One

2. One to many
3. Many to One, and

4. Many to Many

Let's underatand the relationship with real-time examples. for example, one country can have one prime minister(One to One), and a prime minister can have many ministers (One to Many), Also, many MP can have one prime minister and many minister can have many departments.

 Aggregation
Aggregation is a way to achieve Assocaition. Aggregation represents the relationship where on objects as a part of its state. It represents the weak relationship between objects. It is also termed as a has-a relationship in java. Like, inheritence represents the is-a relationship. It another way to reuse object.

 Composition
The composition is also a way to achieve Assocaition. The composition represents the relationship where one object contains other object as a part of its state. There is a strong relationship between the containig object and the depedent object. It is the state where containing objects do not have an independent existence. If you delete the parent object, all the child object will be deleted automatically.

 Advantage of OOPs over Procedure-oriented programming language
1) OOPS makes development and maintenance easier, whereas, in a procedure-oriented programming language, it is not easy to manage if code grows as project size increases.

2) OOPs provides data hiding, whereas, in a procedure-oriented programming language, global data can be accessed from anywhere.

3) OOps provides the ability to stimulates real-world event much more effectively. we can provide the solution of real world problem if we are using object-oriented programming language.
